Analyze the current state of this project and recommend next development priorities.

$ARGUMENTS

## Analysis Steps

### 1. Understand Current State

- Review recent commits and changes
- Check for open issues or TODOs in code
- Identify incomplete features
- Review any roadmap or planning documents

### 2. Assess Work in Progress

- Look for partially implemented features
- Find broken or failing tests
- Identify technical debt
- Check for deprecation warnings

### 3. Identify Opportunities

Apply the planning skill methodology:
- What delivers the most user value?
- What unblocks other work?
- What reduces risk?
- What's the effort vs impact?

## Prioritization Framework

Use RICE scoring where applicable:
- **Reach**: How many users affected?
- **Impact**: How significant is the improvement?
- **Confidence**: How sure are we about the above?
- **Effort**: How much work is required?

## Output Format

Provide prioritized recommendations:

### Immediate (This Sprint)

1. **[Task Name]** - Brief description
   - Why now: Reason for priority
   - Effort: S/M/L
   - Dependencies: Any blockers

### Short-term (Next 2-4 Sprints)

- List of recommended features/fixes

### Long-term Considerations

- Strategic items for future planning

### Risks & Blockers

- Items that could derail progress
- Technical debt requiring attention
- External dependencies

## Notes

- Focus on actionable recommendations
- Consider the project's current stage
- Balance new features with maintenance
- Don't over-plan - provide next 3-5 concrete actions
