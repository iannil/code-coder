### 问题描述

Provider 测试失败，错误信息：
```

```

**原因分析**: Zod v4 内部验证 `_zod.traits.has("$ZodType")` 失败。

### 修复内容

**1. 修复 Log.Level 定义** (`src/util/log.ts:8`)

```typescript
// 之前: 使用 .meta() 方法
export const Level = z.enum(["DEBUG", "INFO", "WARN", "ERROR"])
  .meta({ ref: "LogLevel", description: "Log level" })

// 之后: 使用 .describe() 方法
export const Level = z.enum(["DEBUG", "INFO", "WARN", "ERROR"])
  .describe("Log level (DEBUG, INFO, WARN, ERROR)")
```

**2. 修复测试预加载文件** (`test/preload.ts:76-90`)

测试预加载文件中的 `Log.Level` mock 缺少 `_zod.traits` 属性。

```typescript
// 之前: 简单对象 mock
const mockZodSchema = {
  parse: (v: string) => v,
  safeParse: (v: string) => ({ success: true, data: v }),
  optional: () => mockZodSchema,
  ...
}

// 之后: 使用真实 Zod enum
import z from "zod"
const LogLevel = z.enum(["DEBUG", "INFO", "WARN", "ERROR"])
  .describe("Log level (DEBUG, INFO, WARN, ERROR)")
```

- ✅ TypeScript 编译通过

### 技术洞察

1. **Zod v4 `.meta()` 限制**: `.meta()` 返回的对象在嵌套 object schema 时无法通过内部类型验证

### 新增类型

1. **ConfiguredExecutor** - 配置驱动的执行器
   - 接受 AgentConfig 并使用其 model、temperature、permissions 等
   - 集成权限系统进行工具调用检查
   - 支持基于 risk_threshold 的自动审批
   - 支持 HITL 确认请求

2. **ToolRisk** - 工具风险等级
   - Safe: 只读操作 (read, glob, grep)
   - Low: 代码探索 (codesearch)
   - Medium: 文件修改 (edit, write)
   - High: 外部交互 (bash, shell)
   - Critical: 破坏性操作

### 关键功能

```rust
// 创建配置驱动的执行器
let executor = ConfiguredExecutor::new(config, provider, tools, session_id)
    .with_channel("telegram".to_string())
    .with_user_id("12345".to_string());

// 执行时自动检查权限
let response = executor.execute("help me with code").await?;
```

### 权限检查流程

1. 获取工具权限动作 (Allow/Deny/Ask)
2. 如果 Allow → 直接执行
3. 如果 Deny → 拒绝并返回原因
4. 如果 Ask → 检查是否可自动审批
   - 检查 auto_approve.enabled
   - 检查工具是否在 allowed_tools 列表
   - 检查工具风险是否超过 risk_threshold
   - 检查是否超过 max_approvals
5. 不能自动审批 → 请求用户确认
