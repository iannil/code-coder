## Phase 2 技术总结

### 根本原因: Serde Default vs Derive Default

Rust 的 `#[derive(Default)]` 与 `#[serde(default = "fn")]` 行为不同。`#[serde(default)]` 只在反序列化时生效，而 `Default::default()` 使用类型的默认值。

**受影响模块**: ShellOptions, WriteOptions, ReadOptions, GrepOptions, GlobOptions

**解决方案**: 为所有 Options 结构体实现手动 Default trait

## Phase 3 技术总结

### NAPI 绑定架构

Session 模块通过 NAPI-RS 暴露给 TypeScript:

```rust
// Rust 端 (services/zero-core/src/napi/bindings.rs)
#[napi]
pub struct MessageStoreHandle { ... }

#[napi]
impl MessageStoreHandle {
    pub fn push(&self, message: NapiMessage) -> Result<()> { ... }
    pub fn messages(&self) -> Result<Vec<NapiMessage>> { ... }
}
```

```typescript
// TypeScript 端 (packages/core/src/session.ts)
export class MessageStore implements IMessageStore {
  constructor() {
    if (nativeBindings) {
      this.handle = nativeBindings.createMessageStore()
    }
  }
}
```

### 类型转换层

NAPI 对象与 Rust 类型之间的转换:

| NAPI 类型 | Rust 类型 | 转换方式 |
|-----------|-----------|----------|
| NapiMessage | Message | From/TryFrom trait |
| NapiSessionData | SessionData | From/TryFrom trait |
| MessageStoreHandle | Arc<Mutex<MessageStore>> | NAPI handle 模式 |
| SessionStoreHandle | Arc<Mutex<SessionStore>> | NAPI handle 模式 |

### 降级策略

当原生绑定不可用时，自动降级到 JavaScript 实现:

- `FallbackMessageStore`: 纯内存存储
- `FallbackSessionStore`: Map-based 内存存储 (无 SQLite 持久化)

## Phase 5 技术总结

### Security NAPI 绑定架构

Permission 和 Vault 模块通过 NAPI-RS 暴露给 TypeScript:

```rust
// Rust 端 (services/zero-core/src/napi/bindings.rs)
#[napi]
pub struct PermissionManagerHandle {
    inner: Arc<Mutex<RustPermissionManager>>,
}

#[napi]
impl PermissionManagerHandle {
    pub fn add_rule(&self, rule: NapiPermissionRule) -> Result<()> { ... }
    pub fn grant(&self, permission: NapiPermission) -> Result<()> { ... }
    pub fn check(&self, permission: NapiPermission) -> Result<bool> { ... }
}

#[napi]
pub struct VaultHandle {
    inner: Arc<Mutex<RustVault>>,
}

#[napi]
impl VaultHandle {
    pub fn set(&self, entry: NapiSecretEntry) -> Result<()> { ... }
    pub fn get(&self, name: String) -> Result<Option<NapiSecretEntry>> { ... }
    pub fn save(&self) -> Result<()> { ... }
}
```

```typescript
// TypeScript 端 (packages/core/src/security.ts)
export class PermissionManager implements IPermissionManager {
  constructor() {
    if (nativeBindings) {
      this.handle = nativeBindings.createPermissionManager()
    }
  }
  check(permission: Permission): boolean { ... }
}

export class Vault implements IVault {
  constructor(path: string, password: string) {
    if (nativeBindings) {
      this._handle = nativeBindings.openVault(path, password)
    }
  }
  set(entry: SecretEntry): void { ... }
}
```

### 安全特性

| 特性 | 原生实现 | 降级实现 |
|------|----------|----------|
| Permission 匹配 | 支持通配符 (* prefix/suffix) | 相同逻辑 |
| Vault 加密 | ChaCha20-Poly1305 | 无加密 (仅内存) |
| 密钥派生 | SHA-256 + salt | N/A |
| 文件权限 | Unix 0600 | N/A |

### 降级策略

当原生绑定不可用时，自动降级到 JavaScript 实现:

- `FallbackPermissionManager`: 纯内存权限管理，支持相同的通配符匹配
- `FallbackVault`: Map-based 内存存储 (无加密，仅用于开发/测试)
