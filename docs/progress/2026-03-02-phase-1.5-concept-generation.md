# 能力优先级重构 - 内部能力生成优先 (Phase 1.5)

**日期**: 2026-03-02
**状态**: ✅ 已完成
**任务**: 实现 Evolution Loop Phase 1.5 - 内部能力生成

## 概述

当前 Evolution Loop 在 Phase 1 (内部能力发现) 无匹配时，会直接跳到 Phase 2/3 (学习资源/外部资源)。本次重构添加了 Phase 1.5，在无匹配时优先尝试**生成新的**内部能力，而不是直接使用外部资源。

## 变更流程

```
Phase 1: 内部能力发现 (搜索现有)
    ↓ 无匹配
Phase 1.5: 内部能力生成 (新增) ⭐
    ↓ 生成失败或不适用
Phase 2: 学习资源 (Knowledge/Memory)
    ↓ 无匹配
Phase 3: 外部资源 (Web Search/GitHub/Code Gen)
    ↓ 失败
Phase 4: 自我改进 (Reflection/Sedimentation)
```

## 详细变更

### 1. 新增配置项 (`EvolutionConfig`)

```typescript
// Phase 1.5: Internal Capability Generation
enableConceptGeneration: boolean      // 启用内部能力生成
conceptGenerationMinConfidence: number // 最小置信度阈值 (默认 0.6)
allowedConceptTypes: ConceptType[]    // 允许自动生成的类型
executeGeneratedConcept: boolean      // 生成后立即执行还是仅返回建议
```

默认值:
```typescript
enableConceptGeneration: true,
conceptGenerationMinConfidence: 0.6,
allowedConceptTypes: ["TOOL", "PROMPT", "SKILL", "AGENT", "MEMORY", "HAND", "WORKFLOW"],
executeGeneratedConcept: true,
```

### 2. 新增方法

- `tryConceptGeneration()` - 使用 MetaBuilder 检测能力缺口并生成新概念
- `executeGeneratedConcept()` - 根据概念类型执行生成的概念
- `formatConceptRecommendation()` - 格式化推荐消息

### 3. 扩展类型

- `CapabilitySearchSummary.type` - 新增 `"generation"` 类型
- `EvolutionResult.generatedConcept` - 新增字段记录生成的概念

### 4. 风险控制

- 每个 session 只尝试一次概念生成 (`conceptGenerationAttempted` 标志)
- 类型过滤: 只生成 `allowedConceptTypes` 中的概念
- 置信度阈值: 低于 `conceptGenerationMinConfidence` 的不生成
- 高风险类型 (HAND, WORKFLOW) 需要人工审批 (由 MetaBuilder 控制)
