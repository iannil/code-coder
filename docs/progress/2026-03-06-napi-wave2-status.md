## 功能差距分析

### MCP 模块 (packages/ccode/src/mcp/index.ts)

**当前状态**: 949 行，使用 `@modelcontextprotocol/sdk`

**Native 缺失功能**:

1. **Prompts API**
   - `listPrompts()`, `getPrompt()`
   - ccode 用于 MCP prompt 命令

2. **Resources API**
   - `listResources()`, `readResource()`
   - ccode 用于资源访问

3. **通知处理**
   - `ToolListChangedNotificationSchema`
   - 用于实时工具列表更新

4. **AI SDK 集成**
   - `dynamicTool()`, `jsonSchema()` 转换
   - ccode 将 MCP 工具转为 AI SDK 格式

5. **OAuth 实现差异**
   - ccode: `McpOAuthProvider` + SDK 内置流程
   - Native: `McpOAuthManager` + 自定义流程
   - OAuth 方法虽在 Rust 中定义，但未编译到当前 native module

**重构影响评估**: 高风险 - 可能丢失功能

### LSP 模块 (packages/ccode/src/lsp/server.ts)

**当前状态**: 2,000+ 行

**Native 缺失功能**:

1. **自定义根检测**
   - `NearestRoot()` 函数检测 package.json, deno.json 等
   - 决定使用 Deno LSP vs TypeScript LSP

2. **自动安装**
   - 某些语言服务器可按需安装
   - Native 假设服务器已安装

3. **Bun 进程管理**
   - `BunProc.which()` 用于查找可执行文件
   - Bun 特定的进程 spawn

4. **Archive 处理**
   - `Archive` 模块用于下载/解压语言服务器

**重构影响评估**: 中风险 - 需要保留自定义逻辑
